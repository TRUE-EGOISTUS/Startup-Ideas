# Минимальный бизнес-прототип и тактика выхода на рынок
## Стартап: Гибридная платформа микро-задач и формирования команд для IT-специалистов
## Шаг 1. Сегмент, сценарий и «первая победа»
| Элемент	| Описание |
| ------- | -------- |
| Сегмент	| Начинающие Python-разработчики (Junior, студенты 3–4 курсов, выпускники курсов), активно ищущие практический опыт для портфолио. |
| Сценарий	| Студент заходит на платформу, видит задачу «Разработать API для получения курсов валют», откликается, выполняет, получает отзыв и рейтинг. |
| Ценностное предложение	| «Получи практический опыт, подтверди навыки и найди команду для стартапа — всё на одной платформе.» | 
| «Первая победа»	| Пользователь успешно выполняет первую задачу, получает положительный отзыв, задача отмечена в профиле, рейтинг повышен. Время до победы: 1–3 дня. |

## Шаг 2. Критические рыночные гипотезы (ТОП-3)
### Полный список гипотез (10 шт.)

1. Начинающие разработчики активно ищут возможности для портфолио и готовы регистрироваться.
2. Пользователи понимают ценность подтверждённых задач как доказательства навыков.
3. Студенческие Telegram-чаты — эффективный канал привлечения специалистов.
4. Отсутствие компаний на старте создаёт барьер «курицы и яйца».
5. Разработчики готовы выполнять бесплатные задачи ради опыта и рейтинга.
6. Компании готовы публиковать задачи бесплатно ради доступа к молодым специалистам.
7. Пользователи предпочтут нашу платформу Upwork из-за низкого порога входа.
8. Прямые контакты с HR через LinkedIn/email работают для привлечения компаний.
9. Встроенный чат (long polling) достаточен для коммуникации без внешних мессенджеров.
10. После выполнения задачи пользователи возвращаются за новыми задачами или в команды.

### ТОП-3 гипотезы риска

| №	| Гипотеза	| Риск |
| - | -------- | ---- |
| H1	| Начинающие разработчики активно регистрируются и выполняют задачи без денежного вознаграждения (только ради опыта и портфолио).	| Высокий: если не готовы, то нет активности. |
| H2	| Компании (первые 5–10) соглашаются публиковать задачи бесплатно и давать обратную связь исполнителям.	| Высокий: без задач нет пользователей. |
| H3	| Каналы (студенческие Telegram-чаты) обеспечивают не менее 50 регистраций и 10 выполненных задач за первые 2 недели.	| Средний: недостижение критической массы. |

## Шаг 3. Минимальный бизнес-прототип (МБП)
__Формат:__ Лендинг + форма заявки (concierge).

| Требование	| Описание |
| ---------- | -------- |
| Какой результат демонстрирует	| Лендинг описывает ценность для разработчиков (реальный опыт, отзывы, портфолио) и для компаний (доступ к молодым талантам). Примеры задач и отзывов. |
| Что видит/делает пользователь	| Просматривает лендинг, выбирает задачу из предложенного списка, заполняет форму (имя, email, GitHub). | 
| Целевое действие	| Отправка формы заявки. |
| Какие данные собираем	| Имя, email, ссылка на GitHub, выбранная задача. Отдельная форма для компаний (название, контакт, описание задачи). |

_Механика:_ После заявки команда вручную связывается с разработчиком, выдаёт ТЗ, принимает результат, даёт отзыв. Проверяется готовность выполнять задачи и ценность.

| Доказательный элемент	| «85% студентов МИРЭА заявили, что им не хватает практических задач для портфолио» (результат опроса). |

### Версия A (карьерный фокус)

> _«Преврати учебные проекты в реальный опыт. Получи задачу от компании, выполни её и добавь в резюме. Первые 100 разработчиков получают бесплатный аудит профиля.»_

### Версия B (сообществный фокус)

> _«Хочешь расти в IT? У нас ты найдёшь задачи, отзывы менторов и команду для стартапа. Присоединяйся к сообществу начинающих разработчиков.»_

## Шаг 5. Тактика выхода на рынок: каналы и скрипт
### Выбранные каналы (первая волна)

1. __Telegram-чаты__ студенческих и IT-сообществ (МИРЭА, «IT Беседка», «Код Дурова»).

  * _Почему доступен:_ высокая концентрация ЦА, низкий порог входа.
  * _Доставка сообщения:_ текстовый пост с призывом.
  * _CTA:_ «Заполни форму → [ссылка]».

2. __Прямые контакты с кафедрами вузов__ (или через студенческие союзы) — размещение информации в учебных чатах.

3. __LinkedIn / email рассылка__ по небольшим IT-компаниям и стартапам — предложение опубликовать первую задачу бесплатно.

__Скрипт первого контакта (для Telegram-чата)__

> _«Привет! Я из команды стартапа, который помогает джунам получить реальный опыт. Мы собираем задачи от компаний, а вы можете их выполнить и добавить в портфолио. Первые задачи уже есть: сверстать карточку товара, написать парсер. Всё бесплатно, цель — опыт и отзывы. Кому интересно — заполните форму: [ссылка на лендинг]. Буду рад обратной связи!»_

### Критерии отбора контактов

* Студенты 2–4 курса технических специальностей.
* Начинающие разработчики с GitHub (хотя бы 1 репозиторий).
* Активные участники чатов.

## Шаг 6. Воронка привлечения и активации + метрики
| Шаг	| Событие	| Метрика	| Измерение	| Целевое значение |

__Time-to-value (время до первого успеха):__ 1–3 дня.
